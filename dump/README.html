🔆 Laser Grid Control

A cloud-reachable controller for a 7×7 array of 49 RGB “Laser Space Cannons” driven by Pangolin BEYOND. A browser talks to a small cloud relay; each laser PC dials out to that relay (no port-forwarding) and renders sandboxed pattern code to the lasers over OSC.

1. The big picture

Three moving parts: a droplet (cloud relay + web UI), one or more agents (a Node process on each BEYOND PC), and the mapper (a calibration tool the agent serves on the LAN). Patterns are authored as small JavaScript programs that the droplet ships to the agent and the agent runs in a sandbox, emitting per-cannon colour as BEYOND livecontrol OSC.

Why outbound? The laser PCs sit on networks we don't control (often a phone hotspot). Rather than expose them, each agent makes an outbound WSS connection to the droplet and the relay pushes commands down that pipe. The droplet is the only thing with a public address.

2. Components

Droplet — droplet/relay.js

  • Web UI (single page): solid R/G/B tiles, ~80 live-preview pattern tiles (repo animations + 58 programs from programs.js), grouped Sky Beams and Flags sections (searchable), smiley faces, a colour wheel, and per-tile speed control. Each tile renders a live preview in-browser and, on click, ships a self-contained JS bundle to the agent.
  • Relay: login-gated (cookie session). POST /api/command forwards JSON to the connected agent; GET /api/status reports agent connectivity. The agent connects to /agent?token=AGENT_TOKEN over WSS; the relay keeps a single agent socket and replaces it on a new connection.
  • TLS: Caddy terminates HTTPS using a sslip.io magic-DNS hostname (the droplet IP with dashes, e.g. 203-0-113-9.sslip.io) so no domain registration is needed.
  • Secrets come from env (UI_USER, UI_PASSWORD, AGENT_TOKEN) — see creds.example.txt.

Agent — agent/agent.mjs

  • Dials the droplet (auto-reconnect with capped back-off; pin the droplet IP in hosts if the local DNS is flaky).
  • Runs each pattern in a fresh QuickJS sandbox (engine.mjs + sandbox-prelude.js) with CPU/mem/stack limits — pattern code is treated as untrusted. The pattern fills a 49-cell RGB framebuffer through a ctx API (setHSV/fill/fade/noise/t/dt…).
  • A swappable sink consumes the framebuffer: LocalUiSink (canvas viewer, for dev) or OscSink (drives BEYOND). See §3.
  • Safety limiter (safety.mjs): master brightnessCap and a flash/slew rate cap (maxFlashHz) applied to every frame before output — the sandbox cannot bypass it.
  • Arm gate: OscSink emits nothing until armed:true (or an arm command) — so deploying never disturbs a live show.
  • Hosts the mapper on :mapperPort for direct LAN calibration.

Pattern library — droplet/host.js + droplet/programs.js

UMD modules that run in both the browser (tile previews) and the QuickJS sandbox. host.js exposes createNode/renderFrame; programs.js is a catalogue of ~58 named programs. The droplet serves them at /host.js and /programs.js and bundles their source into the code it ships when you click a program tile.

Mapper — agent/mapper.mjs + agent/mapper.html

Calibration tool the agent serves on the LAN (default :8091/map). It maps each bitmap position (a cell in the 7×7) to a BEYOND zone index, a name (e.g. A1), and a target machine. Two views — a 7×7 grid and an index→name list. Click a cell to flash that cannon for 5 s (real OSC), to identify it. The map persists in mapping.json on the PC (source of truth); the droplet show-control reads the same file. A standalone test build lives in mapper/ (runs without a real BEYOND, logs flashes).

Tools — tools/simulator.html

A standalone in-browser simulator of the 7×7 grid for developing patterns without hardware.

3. OSC addressing (the important details)

BEYOND’s OSC input listens on UDP 8000. Per-zone live control:

/beyond/zone/{N}/livecontrol/alpha        255   = take the zone (0 releases it)
/beyond/zone/{N}/livecontrol/red          0..255
/beyond/zone/{N}/livecontrol/green        0..255
/beyond/zone/{N}/livecontrol/blue         0..255
/beyond/zone/{N}/livecontrol/Brightness   0..100
GotchaDetail
Value scale0..255 floats for RGB/alpha (not 0..1). Sending 1.0 looks “off”. Brightness is 0..100.
Take vs releasealpha=255 to override the running cue; alpha=0 to release back to it.
PacingBEYOND drops bursts — OscSink diffs frames and only sends changed channels, capped per flush.
EnableBEYOND must have OSC input enabled and the R-G-B-A live-control layer active.

4. Topology & multi-PC

Each FB4 DAC is single-master (controlled by exactly one BEYOND, over TCP 3348 on a dedicated laser NIC). With two BEYOND PCs, each drives its own subset of the 49 cannons; they do not stream pixels to each other (BEYOND multi-PC sync is timecode/ArtNet/OSC, not pixel sharing). So to light all 49 from one agent, send OSC to both BEYONDs — that's what osc.targets (e.g. pc1=localhost, pc2=other-IP) and the mapper's per-cell target are for.

5. Deploy

Droplet

# on a fresh Debian/Ubuntu droplet
cp creds.example.txt creds.txt   # fill in
set -a; . ./creds.txt; set +a    # load into env
scp relay.js host.js programs.js root@DROPLET:/root/
bash setup.sh                    # node, Caddy (auto-TLS via sslip),
                                 # systemd unit laser-relay
# UI:  https://<SSLIP>/   (login with UI_USER/UI_PASSWORD)

Agent (per BEYOND PC)

# needs Node 18+ on the PC
cp config.example.json config.json   # fill in relayUrl+token+targets
npm install                          # ws, quickjs-emscripten
node agent.mjs                       # or use run.cmd as a
                                     # SYSTEM startup task
# mapper:  http://<PC-IP>:8091/map
Calibrate with the mapper first (flash each cell → set its zone index + target → Save to device), then arm from the web UI to let patterns drive output.

6. Security

No secrets are in this snapshot. creds.txt, the agent config.json, runtime mapping.json, and node_modules/ are git-ignored. The relay and setup script read all secrets from the environment. Use a strong UI_PASSWORD and a random AGENT_TOKEN — anyone with the token can connect an agent.
